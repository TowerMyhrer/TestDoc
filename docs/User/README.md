# User Guides

Documentation for end users of this Business Central extension.

## About this extension

The **E-Wheels Omnium Integration** connects Microsoft Dynamics 365 Business Central with the Omnium e-commerce and retail platform. It automates the flow of orders, deliveries, inventory updates, and payments between the two systems, reducing manual data entry and keeping both platforms in sync.

## Features

| Feature | Description |
|---------|-------------|
| [Omnium Setup](omnium-setup.md) | One-time configuration page where administrators connect Business Central to Omnium and define global settings for orders, payments, and inventory. |
| [Omnium Stores](omnium-stores.md) | The list of all Omnium stores (physical shops, web warehouses, and external partners), with settings that control how each store is handled in Business Central. |
| [Omnium Log](omnium-log.md) | Audit trail of all integration activity. Use this page to monitor processing status and investigate and retry errors. |
| [Sales Orders from Omnium](sales-orders.md) | Web shop and retail orders from Omnium are automatically created as Sales Orders in Business Central. Also covers automatic credit memos for returns. |
| [Purchase Orders](purchase-orders.md) | Purchase Orders released in Business Central are automatically sent to Omnium for fulfilment. Incoming delivery updates are shown directly on the purchase order. |
| [Omnium Deliveries](omnium-deliveries.md) | Tracks all inbound delivery notifications from Omnium. Covers internal stock transfers, automatic Sales/Purchase Order creation for external store deliveries, and Transfer Order status tracking. |
| [Omnium Order Types and Payment Setup](omnium-order-types.md) | Maps Omnium order type codes to Business Central payment methods and prepayment settings. Also covers the per-store POS payment account mapping. |
| [Item Feed](item-feed.md) | The Base Item flag on items controls which products are exposed to Omnium as top-level catalogue items for the web shop. |
